** Details
*** What version of Xcode are you using?
13.2
** Description
*** Please describe the issue:
The File->New menu contains the commands Editor and Editor Below, both of the which split the editor to provide another editing pain, and can have their keyboard shortcuts configured. But using Editor Below causes the commands to change to Editor and Editor On Right, with the behavior of the Editor command now acting as Editor Below. Invoking Editor On Right causes them to flop back.

When using these commands via keyboard shortcut this is very confusing, since the changing text in the menu items is not seen. Repeatedly invoking the Editor Below/Editor On Right keyboard shortcut causes editors to appear with alternating behavior.
*** What did you expect to happen?
1. Get an editor below.
2. Get an editor on the right.
3. Get an editor below.

Additionally, I would expect the commands to be named in a consistent manner: "Editor On Right" and "Editor Below".
*** What actually happened?
1. Got an editor below.
2. Got an editor on the right.
3. Got an editor below.
